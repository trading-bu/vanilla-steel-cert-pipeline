CHEMICAL ANALYSIS — CRITICAL RULES:

1. ALWAYS identify elements by their COLUMN HEADER, not by position in the row. If a cert lists elements in order "Nb V Ti Cu Cr Ni Mo", map values to those elements exactly: do not reorder them to match a different standard.

2. CEQ / CARBON EQUIVALENT: Some certs (e.g. SSAB) include a "Ceq" or "Carbon equivalent" column that appears near the "C" column. This is NOT the carbon content. Extract Ceq into the "Ceq" field only. Extract actual carbon content (labelled "C") into the "C" field. These are always separate columns — never confuse them. Carbon content C is typically 0.01–0.20% for most steels. Ceq is typically 0.20–0.50%.

3. SCALED INTEGERS: Some certs (e.g. ArcelorMittal) show chemical values as integers with a scale factor in the column header (e.g. "10^-3" or "×10⁻³"). Multiply by the factor to get percentages. Example: if header says "10^-3 %" and value is "72", report C = 0.072. N and B elements sometimes use "10^-4" — check the header for each element.

4. Values are percentages unless clearly labelled otherwise.
5. If the same heat appears multiple times (multiple coils same heat), use the first occurrence.
6. Map chemicals per coil using the heat/cast number.

Common elements to extract: C, Si, Mn, P, S, Al, Cr, Ni, Mo, Cu, V, Ti, Nb, Zr, B, N, Ceq.

MECHANICAL PROPERTIES — extract per heat/coil:

- rm: tensile strength Rm (MPa) — also "UTS", "Tensile Strength"
- a_pct: elongation A% (%) — also "A80", "A50", "Elongation"
- dir: test direction "L" (longitudinal) or "T" (transverse) — default "L"
- bend: cold bend test result (e.g. "Sat", "OK", "Pass")

IMPORTANT RULES:
1. If a field is not found, use null for numbers and "" for strings — never guess or invent values
2. Extract ALL coil rows — do not stop at the first one
3. If chemical values are given per heat and multiple coils share a heat, repeat the same chemistry for each coil
4. Do not include units in numeric fields — numbers only
5. cert_date must be YYYY-MM-DD format regardless of how it appears on the cert
6. For po_number: only extract VS-pattern PO numbers (P0XXXX). Leave blank otherwise.
